## Adapted and derived source code

### Sonolus engine

The play, watch, preview, and tutorial engine began from NonSpicyBurrito's [`sonolus-pjsekai-engine`](https://github.com/NonSpicyBurrito/sonolus-pjsekai-engine), licensed under MIT. The original import did not record an exact upstream revision, so this repository does not claim that the current upstream head is the import source. Haneoka's later adaptations do not remove the upstream notice. It is retained at [`packages/sonolus/engine/LICENSE.pjsekai.txt`](packages/sonolus/engine/LICENSE.pjsekai.txt), and engine build artifacts include both that notice and Haneoka's package `LICENSE`.

### Next-SEKAI

Two separate Next-SEKAI sources apply:

- SUS analysis and conversion semantics in `packages/chart-editor/src/formats/sus.ts` are adapted from Kyle Chang's [`Next-SEKAI/sonolus-next-sekai-engine`](https://github.com/Next-SEKAI/sonolus-next-sekai-engine), MIT; the retained notice is [`packages/chart-editor/LICENSE.next-sekai.txt`](packages/chart-editor/LICENSE.next-sekai.txt).
- Chart-editor toolbar grouping, most-recently-used menu behavior, and portions of flat-timeline interaction geometry are adapted from NonSpicyBurrito's [`Next-SEKAI/sonolus-next-sekai-editor`](https://github.com/Next-SEKAI/sonolus-next-sekai-editor), MIT; the retained notice is [`packages/chart-editor/LICENSE.next-sekai-editor.txt`](packages/chart-editor/LICENSE.next-sekai-editor.txt).

The local imports did not record exact upstream revisions. The notices and source headers identify the repositories and copyright holders; a future import must also record its commit and affected files.

### Unity rendering references and bundled shader programs

The story renderer contains three provenance classes that must not be conflated:

1. `packages/story/src/rendering/post/UnityColorUtils.ts` explicitly ports `UnityEngine.Rendering.ColorUtils` functions, and `UnityColorGrading.ts` implements corresponding color-grading preparation logic. The source family is Unity's [`Unity-Technologies/Graphics`](https://github.com/Unity-Technologies/Graphics) repository, whose repository license is the Unity Companion License. The local files did not record the exact package tag or commit; that version and the applicability of the license to this use must be resolved before redistribution.
2. `packages/story/src/rendering/post/shaders/AdvHdrLutShaders.ts` contains separately identified third-party GLES3 fragment program strings corresponding to Unity `6000.3.12f1` `LutBuilderHdr`; the associated reference record identifies `globalgamemanagers.assets` PathID 113. `AdvColorGradingLut.ts` and `AdvFinalPostFxaa.ts` also document matching program blobs used as fidelity references. These third-party program strings are not Haneoka-authored and are not covered by Haneoka's MPL-2.0 grant.
3. Other ADV and chart post-processing files document behavior matched against serialized game data or shipped URP programs. Unless a file expressly identifies copied upstream source, those comments describe an implementation target; they do not establish a third-party license or a conclusion about copyright status.

The resource-processing and compatibility helpers preserve reproducibility evidence, but reproducibility is not redistribution permission.

## WebGAL interoperability and adapted Terre interface code

The story editor contains MPL-2.0-covered work adapted from both upstream projects:

- [OpenWebGAL/WebGAL](https://github.com/OpenWebGAL/WebGAL), MPL-2.0;
- [OpenWebGAL/WebGAL_Terre](https://github.com/OpenWebGAL/WebGAL_Terre), MPL-2.0.

The July 2026 provenance audit inspected WebGAL at `e7f0abeb855b5b442460743bdaa9778ca751b43f` and Terre at `7b7a2159a5ccead80327437b7305b8fdb47a4e5f`. These commits are the verified audit snapshots. `packages/story-editor/src/formats/webgal.ts` adapts WebGAL command vocabulary, script semantics, escaping, and choice splitting into Haneoka's canonical-project codec. The resource manager, Ribbon, scene tabs, graphical sentence list/cards, command picker, and main editor workspace under `app/` adapt corresponding Terre components to Vue and native ADV semantics.

The complete local-to-upstream mapping, exact source paths, relevant feature commits, and modification descriptions are recorded in [`packages/story-editor/NOTICE.webgal.md`](packages/story-editor/NOTICE.webgal.md). Separate original license copies are retained as [`packages/story-editor/LICENSE.webgal.txt`](packages/story-editor/LICENSE.webgal.txt) and [`packages/story-editor/LICENSE.webgal-terre.txt`](packages/story-editor/LICENSE.webgal-terre.txt). Although their MPL-2.0 texts are byte-identical, both files are kept so neither project's provenance is lost. Executable web builds carry the corresponding copies and [`webgal-and-terre-notice.txt`](public/licenses/webgal-and-terre-notice.txt) under `public/licenses/` together with Haneoka's source-location notice.

The mapped files carry MPL-2.0 headers. Source and executable distributions must preserve the applicable notices and make the corresponding covered Source Code Form available as required by MPL-2.0. Haneoka's modifications to those covered files remain under MPL-2.0; no statement here grants rights in upstream names or trademarks.
